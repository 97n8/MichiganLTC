# Signature Routing

## Purpose
Use this file to clarify who signs which materials, when written addenda are required, and how final approval is documented.

## Routing workflow
1. Identify the document type, purpose, and affected site or workstream.
2. Confirm whether the document is informational, operational, legal, financial, grant-related, regulatory, or external-facing.
3. Confirm the authorized signer or approver based on the current authority map and any written agreements.
4. Route draft materials for review before requesting signature.
5. Record final approval, execution date, and storage location.

## Written addenda required when
- a new site is proposed
- scope expands beyond currently authorized work
- authority changes for sign-off or representation
- new obligations, commitments, or controlled disclosures are introduced
- the repository is asked to support work outside documented authorization

## Final approval documentation
Document final approval with:
- approving authority
- approval date
- version or file name
- related source or governing record
- final storage location
