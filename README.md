# Michigan LTC Network Resource Recovery Corridor

## Project purpose
This repository is the authoritative working structure for the **Michigan LTC Network Resource Recovery Corridor**. It is intended to support governed project execution across funding development, documentation control, site coordination, implementation planning, compliance readiness, training, and related program-support work for the currently authorized project sites in **Coleman** and **Flint, Michigan**.

## Repository status
**Private working repository.** This repository is for internal project operations, controlled collaboration, and audit-ready recordkeeping. It is not a public marketing site and should not be treated as a public source of record for unsupported claims.

## PublicLogic role
PublicLogic supports this project through:
- governance support
- program coordination
- funding coordination
- documentation management
- implementation planning
- training and continuity support
- authorized tooling support

## What this repository is
- A governed workspace for project documentation and decision support
- A structure for organizing grants, governance records, funding materials, site files, implementation planning, and related deliverables
- A place to track sources, routing, approvals, assumptions, and project changes in an audit-ready manner

## What this repository is not
- Not a public-facing marketing website
- Not a place for unsupported claims, informal notes without context, or unverified technical/legal/financial assertions
- Not an approval record unless the relevant authority, source, and routing documentation are included
- Not a storage location for PublicLogic proprietary runtime concepts, reusable tooling, frameworks, methods, or templates beyond what is explicitly authorized for this repository

## Current sites
- Coleman
- Flint

## Operating principles
1. **Source discipline:** Record what is known, where it came from, and what still needs verification.
2. **No unsupported claims:** Do not state awards, endorsements, permits, emissions outcomes, jobs, revenue, investor commitments, or partner approvals without a cited source.
3. **Written-addendum control:** New sites, scope expansion, and authority changes require written documentation and routing.
4. **Clear ownership:** Every deliverable, issue, and decision should have an owner or responsible route.
5. **Audit-ready work:** Materials should be organized so that a reviewer can identify source support, status, and approval history.

## Confidentiality warning
This repository may contain draft, internal, confidential, investor-sensitive, legal, financial, technical, regulatory, and partner-private working materials. Share access only with authorized participants. Do not copy repository contents into public channels or public-facing materials without confirming release authority.

## How to use this repository
1. Start with `/docs`, `/governance`, and `/templates`.
2. Place new working materials in the correct subject-area folder.
3. Use the templates before creating new trackers, notes, or decision records.
4. Record source support in `/docs/source-register/` whenever project claims, requests, or deliverables depend on external documents.
5. Update governance logs when decisions, assumptions, risks, escalations, or changes affect project execution.
6. Keep placeholders in place when source documents are missing rather than inventing content.
