# Source Register

Use this folder to record source documents that support project claims, decisions, deliverables, requests, and submissions.

## Record each source with the following information
- **File name**
- **Received from**
- **Date received**
- **What it supports**
- **Reliability/confidence**
- **Restrictions/confidentiality**
- **Related deliverables**

## Recommended register format

| File Name | Received From | Date Received | What It Supports | Reliability / Confidence | Restrictions / Confidentiality | Related Deliverables |
| --- | --- | --- | --- | --- | --- | --- |
| Placeholder | TBD | TBD | Describe the supported claim, decision, or deliverable | TBD | Note sharing or confidentiality limits | List related outputs |

## Recording guidance
- Create one row per source document or clearly grouped source package.
- Update entries if a source is superseded or clarified.
- If a source is incomplete or informal, record that limitation instead of overstating reliability.
- Link related deliverables, site files, grant materials, or governance records where relevant.
